ApplyGuard ownership correction patch

Changes:
- Removed caller-provided bool ownership decisions from ApplyGuard call sites.
- Added RollbackManager::SaveStateDisposition { CreatedNewState, ReusedExistingState }.
- Changed saveAppliedState() and saveProcessState() to return SaveStateDisposition directly under the rollback mutex.
- Changed ApplyGuard to store rollback state ownership as an enum, not a bool combination.
- Removed BackgroundController hasProcessState() TOCTOU branch from the apply transaction path.
- Changed SchedulerController to always call saveAppliedState(); existing rollback state is preserved and reported as ReusedExistingState.
- ApplyGuard::discardSavedState() now removes rollback state only when the transaction created it.
- ApplyGuard rollback paths now log contextual rollback failures before returning std::unexpected.
- Destructor rollback failure logs now distinguish stale/missing identity, audit failure, and apply failure categories.

Merge blocker addressed:
- Rollback ownership source of truth is now RollbackManager save result, not caller discipline.
- Existence query -> save -> guard TOCTOU in BackgroundController was removed.
