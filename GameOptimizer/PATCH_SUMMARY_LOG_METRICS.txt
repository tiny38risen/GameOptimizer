[PATCH] Log/metrics cleanup

1. ThreadTracker multi-sampling baseline fixed.
   - The first observation in each watchdog window now establishes a local baseline only.
   - Delta accumulation no longer includes the long gap from the previous watchdog cycle.
   - This prevents avg-delta values such as 1000ms+ from appearing under 50ms sampling.

2. Thread log labels clarified.
   - avg-delta -> avg-sample-cpu
   - EMA -> EMA-sample-cpu
   - The values now describe CPU time consumed during sampled intervals, not wall-clock runtime.

3. ICMP transient timeout logging softened.
   - Single missed ICMP sample is INFO, not WARN.
   - Consecutive misses still escalate to WARN and can trigger handle reopen.

4. Background restriction success logs made summary-first.
   - Per-process success logs are suppressed by default.
   - The summary line remains always enabled.
   - Use --background-detail-log to re-enable per-process success lines during diagnosis.

5. Scope control.
   - Modified only ThreadTracker, LatencyMetricsCollector, BackgroundController, and main argument wiring.
